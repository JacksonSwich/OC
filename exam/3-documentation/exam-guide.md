## Как пользоваться шаблоном

> **Важно:** перед запуском `generate-template.py` установи `odfpy`:
> ```bash
> pip3 install odfpy
> ```
> (уже сделано в `setup.sh` на этапе 1)

1. Перейти в папку нужного ГОСТа:
   ```bash
   cd gost-59795-2021    # или cd gost-19.505-79
   ```

2. Скопировать `user-guide-template.md` в новый файл:
   ```bash
   cp user-guide-template.md user-guide-[название-по].md
   ```

3. Заменить все `[скобки]` на конкретику под своё ПО.

4. Раздел 4 (59795) / раздел 3 (19.505-79) — **самый важный**. Нужно описать
   минимум 2-3 реальные операции. Чем детальнее — тем выше балл.

5. Если времени мало — можно сократить количество операций,
   но каждую описать по всем пунктам.

## Ответы на защиту

### Вопрос: Почему выбрали именно этот ГОСТ?

**Ответ:** ГОСТ Р 59795-2021 — действующий национальный стандарт РФ,
который устанавливает требования к содержанию документов на
автоматизированные системы. Он заменил более старые ГОСТ 34 серии.
Документ «Руководство пользователя» предусмотрен разделом 6.4 этого ГОСТ.

### Вопрос: Какие разделы必обязательно должны быть в руководстве?

**Ответ:** Шесть разделов:
1. Введение
2. Назначение и условия применения
3. Подготовка к работе
4. Описание операций (самый объёмный)
5. Аварийные ситуации
6. Рекомендации по освоению

### Вопрос: Почему раздел «Описание операций» самый важный?

**Ответ:** Потому что именно он показывает, как пользователь будет реально
работать с программой. По ГОСТ 6.4.6 для каждой операции нужно указать:
наименование, условия выполнения, подготовительные действия, основные
действия (пошагово), заключительные действия и ресурсы. Это доказывает,
что программа действительно автоматизирует какие-то функции.

### Вопрос: Чем Linux отличается от Windows в документе?

**Ответ:** Только установкой (раздел 3.2) и командами аварийного
восстановления (раздел 5). В Linux — пакетные менеджеры (apt, dpkg),
консольные команды kill/journalctl. В Windows — установщики .exe/.msi,
taskkill. Сама логика работы с программой одинакова.

### Вопрос: Что писать в «Аварийные ситуации», если программа стабильна?

**Ответ:** Всё равно нужно описать хотя бы общие сценарии: зависание
программы, потеря сетевого соединения (если нужно), и как восстановить
данные из бэкапа. Это обязательный раздел по ГОСТ 6.4.7, его отсутствие
снизит баллы.

### Вопрос: Можно ли объединять разделы?

**Ответ:** Можно. По ГОСТ раздел 4.2 допускает объединение и исключение
разделов в зависимости от особенностей системы. Но лучше не сокращать,
а просто писать короче, чтобы все 6 разделов были в документе.

### Ответы на защиту по ГОСТ 19.505-79

**Вопрос: Чем отличается ГОСТ 19.505-79 от ГОСТ Р 59795-2021?**

**Ответ:** ГОСТ 19.505-79 — это более старый стандарт ЕСПД (Единой системы программной документации), который определяет «Руководство оператора». Он содержит 4 обязательных раздела вместо 6. Главное отличие — нет отдельного раздела «Аварийные ситуации» (информация о сбоях включается в раздел «Сообщения оператору») и нет «Рекомендаций по освоению». Раздел «Выполнение программы» объединяет установку, запуск, операции и завершение.

**Вопрос: Какие разделы обязательны по ГОСТ 19.505-79?**

**Ответ:** Четыре раздела (п. 1.2):
1. Назначение программы
2. Условия выполнения программы
3. Выполнение программы
4. Сообщения оператору

Также обязательны аннотация и содержание (по ГОСТ 19.105-78, п. 1.1).

**Вопрос: Что должно быть в разделе «Выполнение программы»?**

**Ответ:** По п. 2.3 ГОСТ 19.505-79 — последовательность действий оператора, обеспечивающих загрузку, запуск, выполнение и завершение программы, описание функций, формата и вариантов команд, с помощью которых оператор управляет программой, а также ответы программы на эти команды.

**Вопрос: Где описывать аварийные ситуации в ГОСТ 19.505-79?**

**Ответ:** В разделе «Сообщения оператору» (п. 2.4). Там должны быть приведены тексты сообщений, выдаваемых в ходе выполнения программы, описание их содержания и соответствующие действия оператора (включая действия при сбое и возможность повторного запуска).

**Вопрос: Можно ли объединять разделы в ГОСТ 19.505-79?**

**Ответ:** Да, п. 1.2 допускает объединение отдельных разделов или введение новых в зависимости от особенностей программы.

## Быстрый шаблон для раздела «Описание операций» / «Выполнение программы»

### Для ГОСТ Р 59795-2021 (раздел 4)

Каждую операцию пиши по этой схеме:

```
### Операция: [Название]

**Наименование:** [название]

**Условия:**
- [условие 1]
- [условие 2]

**Подготовка:**
1. [действие]
2. [действие]

**Основные действия:**
1. [действие]
2. [действие]
3. [действие]

**Завершение:**
- [действие]

**Ресурсы:** время — [..], память — [..]
```

### Для ГОСТ 19.505-79 (раздел 3)

Та же структура, но добавляется пункт «Ответ программы на команды»:

```
### Операция: [Название]

**Наименование:** [название]

**Условия:**
- [условие 1]

**Подготовительные действия:**
1. [действие]

**Основные действия:**
1. [действие]
2. [действие]

**Заключительные действия:**
- [действие]

**Ресурсы:** время — [..], память — [..]

**Ответ программы на команды:**
- при успехе: [описание]
- при ошибке: [описание]
```

## Быстрая работа на экзамене

### Какой ГОСТ использовать?

1. **Посмотреть в билете:** если написано «ГОСТ 19.505-79*» — иди в `gost-19.505-79/`, иначе в `gost-59795-2021/`.
2. **Внутри папки** файлы называются одинаково: `user-guide-template.md` и `generate-template.py`.

### Скрипт для быстрой подстановки

```bash
# Переходим в нужную папку
cd gost-59795-2021        # или cd gost-19.505-79

# Копируем шаблон
cp user-guide-template.md user-guide-final.md

# Открываем в nano и заменяем все [скобки] через поиск
nano user-guide-final.md

# → Ctrl+W, вводим "[", жмём Enter
# → Заменяем на нужное значение
# → Повторяем, пока не закончатся скобки
```

### Генерация .odt

```bash
# Из папки ГОСТа:
python3 generate-template.py "Название ПО" "user-guide-название.odt"
```
