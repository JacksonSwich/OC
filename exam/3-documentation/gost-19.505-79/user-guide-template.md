## 3. Выполнение программы

### 3.1 Загрузка программы

#### Linux (Ubuntu)

**Способ 1 — установка из репозитория:**
```bash
# Добавление репозитория (если требуется)
echo "deb [репозиторий]" | sudo tee /etc/apt/sources.list.d/[файл].list
# Импорт ключа GPG
wget -qO- [ключ] | sudo apt-key add -
# Установка
sudo apt update
sudo apt install [пакет]
```

**Способ 2 — установка из скачанного пакета:**
```bash
# Для .deb
sudo dpkg -i [файл].deb
# Если есть зависимости
sudo apt --fix-broken install

# Для .AppImage
chmod +x [файл].AppImage
./[файл].AppImage
```

#### Windows

**Способ 1 — установка через установщик:**
1. Скачать установочный файл с официального сайта.
2. Запустить `[файл].exe` / `[файл].msi` от имени администратора.
3. Следовать инструкциям мастера установки.

**Способ 2 — установка через Chocolatey (при наличии):**
```powershell
choco install [пакет]
```

**Проверка загрузки:**
После установки необходимо проверить, что программа установлена корректно.

**Linux:**
```bash
# Проверка версии
[команда] --version

# Запуск программы
[команда запуска]
```

**Windows:**

Если программа запустилась и отображает основной интерфейс — установка выполнена успешно.

### 4.3 Действия оператора при сбоях

При возникновении сбоя в процессе выполнения программы необходимо:

1. Зафиксировать текст сообщения об ошибке.
2. Проверить сетевое подключение (если требуется).
3. Убедиться, что сервер (если требуется) доступен и работает.

**Проверка системных журналов:**
- **Linux:** `journalctl -xe` или `tail -f /var/log/syslog`
- **Windows:** `eventvwr.msc` (Журнал событий)

### 4.4 Восстановление программы после сбоя

После устранения причины сбоя необходимо:

1. Закрыть программу (см. п. 3.4).
2. Проверить целостность данных:
   - **Linux:** `cp -r ~/.config/[программа] ~/backup/`
   - **Windows:** копировать папку `%APPDATA%\[программа]` в резервную папку.
3. Запустить программу повторно (см. п. 3.2).
4. Если программа не запускается — переустановить её (см. п. 3.1).
5. При необходимости восстановить данные из резервной копии.

### 4.5 Возможность повторного запуска программы

После устранения сбоя программа может быть запущена повторно в обычном режиме (см. п. 3.2). Если сбой был вызван ошибками в данных, необходимо восстановить данные из резервной копии перед запуском.

При обнаружении признаков несанкционированного доступа:
1. Немедленно завершить работу программы.
2. Сменить пароль учётной записи.
3. Проверить журнал доступа (логи).
4. Сообщить администратору системы.
